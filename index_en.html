Hi, welcome

Welcome to Pablo Alonso García's personal website. Here you'll find some information about my professional profile. I write a blog, where I cover topics like software development, agile methodologies, book reviews, and I write some tutorials about web technologies.

What should you know about me?

I'm a software and web developer

Since I first got in touch with the HTML language I haven't stop using it. What I enjoy the most about the web is that you can create apps that can be accessed by users all around the globe. So web development is not only my passion but I can make a living out of it :-). By the way, I have a B.S. in Computer Science and a Master Degree in Web Engineering.

I always try to apply best practices to ensure the quality of my work, like: prototyping, source code versioning, automatic deploys, test driven development, refactoring, pair programming.

If you want to take a look at some of the code I wrote, you can visit my GitHub profile.

I'm working with Ruby On Rails lately

I've taken part in several projects, including various languages, like Java, Javascript and XUL, PHP. In 2007 I discovered Ruby On Rails: the open-source web application framework. Ruby is a funny language, and Rails provides solutions to the most common problems in the web environment. And there's a great community of people who really like to write good and clean code. Those are the main reasons why I've chosen RoR as my current environment.

I believe in agile methodologies

I don't believe in the waterfall model anymore. After some working experiences, I can say that it doesn't work. I believe in the principles of the Agile Manifesto. Methodologies like SCRUM are hitting bit, and are working really well, so I'm trying to apply them in every project I take part in. Of course, I still have too many things to learn, and that's why I attend events like Agile Spain 2010.

I believe in small companies

I've worked in big companies. However, I'm sure there's a better way to work. I know that a small team of brilliant and truly motivated people can fight against the big players. And, what it's more important: they can develop profitable apps that meet the client's needs and are useful to its users. And all that can be achieved in a good and confident atmosphere.

I think about the users

As a computer scientist, I've participated in projects where we worked for long periods of time with no feedback from the users. What do you want your product for if you have no users, or if your users don't know how to use it? I now believe in the release early, release often paradigm, and I understand how important it is to take users serious at the time of designing a web application.

Is that all?

If you want more info, please visit my LinkedIn profile and my blog. Thanks for passing by!

Pablo.
